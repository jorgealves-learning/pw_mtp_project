## 1Ô∏è‚É£ Issues and Discussions

**Purpose:** Track actionable tasks and discuss ideas.  

**Practical Examples:**

| Action | Example |
|--------|---------|
| **Reporting a bug** | **Title:** Fix login redirect issue <br> **Description:** Users are redirected to the homepage instead of dashboard after login. <br> **Labels:** bug <br> **Assignee:** @username <br> **Real Example:** [React Issue #35101](https://github.com/facebook/react/issues/35101) |
| **Proposing a feature** | **Title:** Add dark mode option <br> **Description:** Implement dark mode for the website to improve user experience. <br> **Labels:** enhancement <br> **Assignee:** @username |
| **Opening a discussion** | **Topic:** Brainstorm new dashboard layout <br> **Content:** Discuss ideas for improving the main dashboard interface. <br> **Real Example:** [Acme App Discussion #29](https://github.com/GELLIFY/acme-app/discussions/29) |

## 2Ô∏è‚É£ Pull Requests and Reviews

**Purpose:** Propose changes and collaborate on code/content improvements.  

**Practical Examples:**

| Action | Example |
|--------|---------|
| **Creating a pull request** | **Title:** Add section on GitHub Actions <br> **Branch:** feature/github-actions <br> **Description:** Document how to set up GitHub Actions for CI/CD. <br> **Real Example:** [Rust PR #148829](https://github.com/rust-lang/rust/pull/148829) |
| **Reviewing a PR** | Leave comments suggesting edits, improvements, or approve changes. <br> **Real Example:** [Rust PR #148829](https://github.com/rust-lang/rust/pull/148829) |
| **Linking PR to Issue** | In the PR description, reference the Issue: `Closes #12` |

**Practical Examples:**

| Action | Example |
|--------|---------|
